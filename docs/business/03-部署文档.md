# 部署文档

更新时间：`2026-04-03`

本文件面向现场实施人员。目标是：从空环境出发，把项目部署到客户宿主机，并把可运行的 skill 投放到 OpenClaw workspace，使 OpenClaw 能识别并调用当前业务能力。

## 1. 交付目标

部署完成后，现场环境至少需要满足：

1. 可以安装指定 tag 或 latest 版本的项目。
2. 可以在没有预装 Python / uv / Playwright 的情况下完成初始化。
3. 可以生成项目运行所需的浏览器配置和运行目录。
4. 可以把可运行 skill 部署到 OpenClaw 默认 workspace。
5. 可以完成环境级 smoke check。

说明：

- 这里验证的是“从 0 到可运行环境”。
- 这里不要求在部署脚本里执行真实业务抓取或真实写回。

## 2. 交付物

实施人员最终会使用：

- `examples/openclaw/deploy-openclaw.sh`
- `examples/openclaw/deploy-openclaw.ps1`
- `skills/mujitask-tiktok-feishu-sync/`

默认网页下载地址：

- `https://raw.githubusercontent.com/knighterrantsky/mujitask/<release-tag>/examples/openclaw/deploy-openclaw.sh`
- `https://raw.githubusercontent.com/knighterrantsky/mujitask/<release-tag>/examples/openclaw/deploy-openclaw.ps1`

## 3. 支持宿主机

- macOS
- Windows

## 4. 部署脚本负责完成的事情

部署脚本固定负责：

1. 检测宿主机系统。
2. 检测并安装缺失依赖。
3. 交互式采集部署所需信息。
4. 下载并解压源码。
5. 下载并安装 pinned `automation-framework`。
6. 安装项目依赖并创建 `.venv`。
7. 安装 Playwright Chromium。
8. 检测本机 Chrome。
9. 生成项目浏览器配置。
10. 生成 skill 本地配置。
11. 投放 skill 到 OpenClaw workspace。
12. 执行环境级 smoke check。

## 5. 交互式输入项

部署脚本需要采集：

- `repo_url`
- `tag`
- `install_dir`
- `table_url`
- `FEISHU_ACCESS_TOKEN`

规则：

- `repo_url` 可以是任意可访问的仓库 URL。
- 如果 `repo_url` 是 GitHub 仓库：
  - `tag` 有值时下载该 tag 压缩包。
  - `tag` 为空时自动解析 latest。
- 如果 `repo_url` 不是 GitHub 仓库：
  - 额外要求输入一个可访问的 `archive_url`。

说明：

- OpenClaw workspace 路径不作为输入项。
- 部署脚本按宿主机系统自动使用默认 workspace。

## 6. 依赖准备策略

部署脚本必须假设目标机可能没有：

- `uv`
- Python `3.11`
- Playwright Chromium
- 项目目录

因此部署脚本需要：

- 缺 `uv` 时自动安装 `uv`
- 执行 `uv python install 3.11`
- 用 `uv venv --python 3.11` 创建虚拟环境
- 不依赖本机预装 `git`

说明：

- 业务仓库源码采用压缩包下载。
- `automation-framework` 也采用根据 manifest 下载源码压缩包并本地安装的方式，避免依赖 Git 源安装。

## 7. 浏览器策略

当前浏览器策略固定为：

- `profile_ref = local-chrome`
- `provider = chrome_cdp`
- `debug_http = http://127.0.0.1:9222`

部署脚本必须：

1. 检测本机是否已安装 Chrome。
2. 如果没有 Chrome，只提示“请先安装 Chrome，然后重新执行部署脚本”。
3. 如果有 Chrome，生成：
   - `config/browser_profiles.json`
   - 浏览器 CDP 启动脚本

说明：

- skill 包中的 `run_batch_sync.*` 会先检查 `http://127.0.0.1:9222`。
- 如果 CDP 未启动，会自动尝试调用 `start_browser_cdp.*`。

## 8. 生成的项目配置

部署脚本需要在项目目录生成或刷新：

- `config/browser_profiles.json`
- `runtime/cli_runs/`
- `runtime/artifacts/`
- `runtime/downloads/`

说明：

- 项目目录不持久化飞书 token。
- token 只持久化到已部署 skill 的本地配置文件中。

## 9. Skill 投放

部署脚本必须把 skill 包投放到 OpenClaw 默认 workspace：

- macOS: `~/.openclaw/workspace/skills`
- Windows: `%USERPROFILE%\.openclaw\workspace\skills`

部署后的目标目录：

- `<openclaw_workspace>/mujitask-tiktok-feishu-sync/`

该目录至少需要包含：

- `SKILL.md`
- `skill.local.env`
- `run_cleanup.*`
- `run_batch_sync.*`
- `start_browser_cdp.*`

## 10. Token 持久化方式

飞书 token 不写入需求文档，不写入仓库业务说明文档，不写入 shell rc。

唯一持久化位置：

- 已部署 OpenClaw skill 目录下的 `skill.local.env`

本地配置项只保留：

- `INSTALL_DIR`
- `TABLE_URL`
- `FEISHU_ACCESS_TOKEN`

## 11. 环境级 Smoke Check

部署脚本完成后，只做环境级验证：

1. 项目 venv CLI 存在。
2. `automation-business-scaffold-run list-tasks` 可以执行。
3. 任务列表中存在：
   - `tiktok_product_link_cleanup`
   - `tiktok_feishu_batch_sync`
4. OpenClaw workspace 中 skill 目录存在且文件完整。

说明：

- 这里不执行真实 `draft` / `canary` 业务写回。
- 这里不在部署脚本里验证客户真实表格抓取结果。

## 12. 实施完成判定

只有同时满足下面条件，才算现场实施完成：

- 指定 tag 或 latest 版本已成功安装。
- 宿主机缺失依赖已补齐。
- 项目 `.venv` 已创建。
- `config/browser_profiles.json` 已生成。
- Chrome 已存在，或已经明确提示安装后重跑。
- OpenClaw workspace 中已经出现 `mujitask-tiktok-feishu-sync`。
- skill 本地配置 `skill.local.env` 已生成。
- `list-tasks` 已成功，且能看到两个正式业务入口。
