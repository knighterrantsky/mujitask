# 部署文档

更新时间：`2026-04-10`

本文件面向现场实施人员。目标是：从空环境出发，把项目部署到客户宿主机，并把可运行的 skill 投放到 OpenClaw workspace，使 OpenClaw 能识别并调用当前业务能力。

## 1. 交付目标

部署完成后，现场环境至少需要满足：

1. 可以安装指定 tag 或 latest 版本的项目。
2. 可以在没有预装 Python / uv / Playwright 的情况下完成初始化。
3. 可以生成项目运行所需的浏览器配置和运行目录。
4. 可以把可运行 skill 部署到 OpenClaw 默认 workspace。
5. 可以完成环境级 smoke check。

说明：

- 这里验证的是“从 0 到可运行环境”。
- 这里不要求在部署脚本里执行真实业务抓取或真实写回。

## 2. 交付物

实施人员最终会使用：

- `examples/openclaw/deploy-openclaw.sh`
- `examples/openclaw/update-openclaw.sh`
- `examples/openclaw/deploy-openclaw.ps1`
- `examples/openclaw/verify-openclaw.sh`
- `examples/openclaw/verify-openclaw.ps1`
- `skills/mujitask-tiktok-feishu-sync/`

默认网页下载地址：

- `https://raw.githubusercontent.com/knighterrantsky/mujitask/<release-tag>/examples/openclaw/deploy-openclaw.sh`
- `https://raw.githubusercontent.com/knighterrantsky/mujitask/<release-tag>/examples/openclaw/deploy-openclaw.ps1`

推荐的一键下载安装方式：

macOS：

```bash
RELEASE_TAG="<release-tag>" # 例如 v3.0.0
curl -fsSL "https://raw.githubusercontent.com/knighterrantsky/mujitask/${RELEASE_TAG}/examples/openclaw/deploy-openclaw.sh" -o deploy-openclaw.sh
chmod +x deploy-openclaw.sh
./deploy-openclaw.sh
```

Windows：

```powershell
$ReleaseTag = "<release-tag>" # 例如 v3.0.0
Invoke-WebRequest "https://raw.githubusercontent.com/knighterrantsky/mujitask/$ReleaseTag/examples/openclaw/deploy-openclaw.ps1" -OutFile deploy-openclaw.ps1
powershell -ExecutionPolicy Bypass -File .\deploy-openclaw.ps1
```

说明：

- 把 `<release-tag>` 替换成目标发布版本，例如 `v3.0.0`。
- `deploy-openclaw.sh` 用于首装。
- `update-openclaw.sh` 用于升级。

## 3. 支持宿主机

- macOS
- Windows

补充说明：

- 更新方案 v1 当前只支持 macOS 的 `update-openclaw.sh`。
- Windows 仍沿用原部署路径，升级专用入口留待后续补齐。

## 4. 首装脚本负责完成的事情

`deploy-openclaw.sh` / `deploy-openclaw.ps1` 固定负责：

1. 检测宿主机系统。
2. 检测并安装缺失依赖。
3. 交互式采集部署所需信息。
4. 下载并解压源码。
5. 从 `.platform/platform-manifest.yaml` 读取并安装 pinned `automation-framework`。
6. 安装项目依赖并创建 `.venv`。
7. 安装 Playwright Chromium。
8. 检测本机 Chrome。
9. 生成项目浏览器配置。
10. 生成 skill 本地配置。
11. 投放 skill 到 OpenClaw workspace。
12. 执行环境级 smoke check。

## 5. 首装交互式输入项

部署脚本需要采集：

- `repo_url`
- `github_pat`（可选）
- `tag`
- `install_dir`
- `table_url`
- `FEISHU_ACCESS_TOKEN`

规则：

- `repo_url` 可以是任意可访问的仓库 URL。
- `github_pat` 只在 GitHub 私有仓库场景下需要，公开仓库可直接留空。
- 部署脚本会先读取宿主机环境变量 `GITHUB_TOKEN` 或 `GH_TOKEN`。
- 只有在本地环境变量不存在时，脚本才会提示手工输入 `github_pat`。
- 如果本地已经存在同一个 `install_dir` 的历史部署：
  - `repo_url` 会优先从 `runtime/deployment/openclaw-deploy.env` 自动复用。
  - `table_url` 和 `FEISHU_ACCESS_TOKEN` 会优先从已部署 skill 的 `skill.local.env` 自动复用。
  - 自定义 `archive_url` 也会优先从历史部署状态自动复用。
- 只有新增字段或本地缺失字段，部署脚本才会再次提示输入。
- 如果 `repo_url` 是 GitHub 仓库：
  - `tag` 有值时下载该 tag 压缩包。
  - `tag` 为空时自动解析 latest。
- 如果 `repo_url` 不是 GitHub 仓库：
  - 额外要求输入一个可访问的 `archive_url`。

说明：

- OpenClaw workspace 路径不作为输入项。
- 部署脚本按宿主机系统自动使用默认 workspace。

## 6. 依赖准备策略

部署脚本必须假设目标机可能没有：

- `uv`
- Python `3.11`
- Playwright Chromium
- 项目目录

因此部署脚本需要：

- 缺 `uv` 时自动安装 `uv`
- 执行 `uv python install 3.11`
- 用 `uv venv --python 3.11` 创建虚拟环境
- 不依赖本机预装 `git`

说明：

- 业务仓库源码采用压缩包下载。
- `automation-framework` 固定从 `.platform/platform-manifest.yaml` 读取 `framework_repo_url` 和 `framework_commit`，再下载源码压缩包并本地安装。
- 如果业务仓库或 `automation-framework` 仓库是 GitHub 私有仓库，必须提供可读权限的 GitHub PAT。

## 7. 浏览器策略

当前浏览器策略固定为：

- `profile_ref = local-chrome`
- `provider = chrome_cdp`
- `debug_http = http://127.0.0.1:9222`

部署脚本必须：

1. 检测本机是否已安装 Chrome。
2. 如果没有 Chrome，只提示“请先安装 Chrome，然后重新执行部署脚本”。
3. 如果有 Chrome，生成：
   - `config/browser_profiles.json`
   - 浏览器 CDP 启动脚本

说明：

- 需要浏览器的 step 脚本会先检查 `http://127.0.0.1:9222`。
- 如果 CDP 未启动，会自动尝试调用 `start_browser_cdp.*`。

## 8. 生成的项目配置

首装脚本需要在项目目录生成或刷新：

- `config/browser_profiles.json`
- `runtime/cli_runs/`
- `runtime/artifacts/`
- `runtime/downloads/`
- `runtime/deployment/openclaw-deploy.env`

说明：

- 项目目录不持久化飞书 token。
- token 只持久化到已部署 skill 的本地配置文件中。
- `runtime/deployment/openclaw-deploy.env` 只保存部署元数据，例如：
  - `REPO_URL`
  - `LAST_RESOLVED_REF`
  - 自定义源码压缩包地址
  - `INSTALL_LAYOUT_VERSION=1`
  - `UPDATE_SUPPORTED=1`

## 9. Skill 投放

部署脚本必须把 skill 包投放到 OpenClaw 默认 workspace：

- macOS: `~/.openclaw/workspace/skills`
- Windows: `%USERPROFILE%\.openclaw\workspace\skills`

部署后的目标目录：

- `<openclaw_workspace>/mujitask-tiktok-feishu-sync/`

说明：

- OpenClaw workspace 下的 skill 目录采用直接替换策略
- 不保留 `mujitask-tiktok-feishu-sync.backup-*`
- 只有项目安装目录允许使用 backup 策略
- `SKILL.md` 必须带 YAML frontmatter，且至少包含 `name` 和 `description`

该目录至少需要包含：

- `SKILL.md`
- `skill.local.env`
- `run_cleanup_step.sh`
- `run_pending_rows_step.sh`
- `run_single_row_update_step.sh`
- `run_keyword_candidate_step.sh`
- `run_insert_seed_row_step.sh`
- `run_fastmoss_login_check_step.sh`
- `start_browser_cdp.*`

## 10. Token 持久化方式

飞书 token 不写入需求文档，不写入仓库业务说明文档，不写入 shell rc。

唯一持久化位置：

- 已部署 OpenClaw skill 目录下的 `skill.local.env`

首装脚本固定写入：

- `INSTALL_DIR`
- `TABLE_URL`
- `FEISHU_ACCESS_TOKEN`

如果 workspace skill 目录中已存在额外本地配置项，例如：

- `BROWSER_PROFILE_REF`
- `FASTMOSS_PHONE`
- `FASTMOSS_PASSWORD`

则升级时必须保留，不得覆盖删除。

## 11. 环境级 Smoke Check

部署脚本完成后，只做环境级验证：

1. 项目 venv CLI 存在。
2. `automation-business-scaffold-run list-tasks` 可以执行。
3. 任务列表中存在：
   - `tiktok_product_link_cleanup`
   - `feishu_pending_rows_scan`
   - `feishu_single_row_update`
   - `feishu_seed_row_insert`
   - `fastmoss_keyword_candidate_discovery`
   - `fastmoss_login_check`
4. OpenClaw workspace 中 skill 目录存在且文件完整。
5. OpenClaw workspace 中不存在旧的 skill backup 目录。
6. 当前 skill 目录中的 `SKILL.md` frontmatter 完整，且 step 脚本齐全。

说明：

- 这里不执行真实业务写回。
- 这里不在部署脚本里验证客户真实表格抓取结果。

## 12. 部署后验证脚本

工程内提供两套部署后验证脚本：

- `examples/openclaw/verify-openclaw.sh`
- `examples/openclaw/verify-openclaw.ps1`

用途：

- 自动从默认 OpenClaw workspace 读取 `skill.local.env`
- 自动解析 `INSTALL_DIR`
- 检查 CLI、任务列表、浏览器配置、部署状态文件、skill 包完整性
- 检查部署状态是否声明 `INSTALL_LAYOUT_VERSION=1` 和 `UPDATE_SUPPORTED=1`
- 检查 `SKILL.md` frontmatter 是否包含 OpenClaw 识别所需的 `name` 与 `description`
- 检查 `http://127.0.0.1:9222` 是否可用

macOS：

```bash
bash examples/openclaw/verify-openclaw.sh
```

Windows：

```powershell
powershell -ExecutionPolicy Bypass -File .\examples\openclaw\verify-openclaw.ps1
```

如果 skill 不在默认 workspace，也可以显式传入 skill 目录：

macOS：

```bash
bash examples/openclaw/verify-openclaw.sh "/Users/yourname/.openclaw/workspace/skills/mujitask-tiktok-feishu-sync"
```

Windows：

```powershell
powershell -ExecutionPolicy Bypass -File .\examples\openclaw\verify-openclaw.ps1 -SkillDir "$env:USERPROFILE\.openclaw\workspace\skills\mujitask-tiktok-feishu-sync"
```

## 13. 实施完成判定

只有同时满足下面条件，才算现场实施完成：

- 指定 tag 或 latest 版本已成功安装。
- 宿主机缺失依赖已补齐。
- 项目 `.venv` 已创建。
- `config/browser_profiles.json` 已生成。
- Chrome 已存在，或已经明确提示安装后重跑。
- OpenClaw workspace 中已经出现 `mujitask-tiktok-feishu-sync`。
- OpenClaw workspace 中不存在旧的 `mujitask-tiktok-feishu-sync.backup-*`。
- skill 本地配置 `skill.local.env` 已生成。
- skill 的 `SKILL.md` frontmatter 与 step 脚本已就绪。
- `list-tasks` 已成功，且能看到两个正式业务入口。

## 14. 后续版本升级

更新方案 v1 当前采用“首装脚本 + 升级脚本”拆分模型：

- `deploy-openclaw.sh`：只负责首装
- `update-openclaw.sh`：只负责升级

升级规则：

1. 不再重复执行 `deploy-openclaw.sh` 做升级。
2. `update-openclaw.sh` 只支持由新版 `deploy-openclaw.sh` 首装出来的环境。
3. 升级过程保留原 `install_dir` 下的：
   - `.venv/`
   - `runtime/`
   - `.env`
   - `config/browser_profiles.json`
4. 升级后会重新投放 OpenClaw workspace skill，但会 merge 保留旧 `skill.local.env` 中的扩展键。
5. 当前 v1 不提供自动备份、自动回滚，也不接管历史旧安装。

macOS 升级步骤：

1. 确认当前环境是由新版 `deploy-openclaw.sh` 安装完成。
2. 从目标版本重新下载：
   - `examples/openclaw/update-openclaw.sh`
3. 执行升级脚本。
4. 交互项只保留：
   - `tag`：填写新的目标版本；如果希望直接安装最新版本，可留空自动解析 latest
   - `github_pat`：仅在 GitHub 私有仓库且本机未设置 `GITHUB_TOKEN` / `GH_TOKEN` 时输入
5. 等待脚本完成环境级 smoke check。

升级结果：

- 原 `install_dir` 不会被删除。
- `runtime/cli_runs`、`runtime/artifacts`、`runtime/downloads` 会原样保留。
- 现有 `config/browser_profiles.json` 不会被覆盖。
- OpenClaw workspace 中的 skill 会被重新投放。
- `skill.local.env` 中的 `INSTALL_DIR`、`TABLE_URL`、`FEISHU_ACCESS_TOKEN` 会按当前值写回。
- `BROWSER_PROFILE_REF`、`FASTMOSS_PHONE`、`FASTMOSS_PASSWORD` 以及其他未知扩展键会被保留。
