# 需求文档

更新时间：`2026-04-15`

## 1. 文档目的

本文件只描述客户需求本身，包括客户当前使用的飞书表、业务目标、信息采集方式、交付形式和验收口径。  
具体的程序设计、模块设计、接口设计、调度实现、浏览器实现和登录实现，不在本文件中展开，统一放到设计文档说明。

## 2. 客户背景与业务目标

客户当前的核心诉求是持续收集 TK 竞品信息，并将结果写入客户现有的飞书多维表中。

客户希望支持两种采集方式：

1. 客户在飞书多维表中维护商品链接，OpenClaw 每天定时抓取竞品数据，并写回飞书多维表。
2. 客户在 OpenClaw 中输入类似“帮我查询/收集关键字为 `east egg` 的 7 日内销量大于 200 的 TK 商品数据”这样的自然语言指令，OpenClaw 自动搜索竞品并写入飞书多维表。

## 4. 客户需求

### 4.1 需求一：飞书表内已有商品的定时更新

客户会先在飞书多维表中添加对应商品的 `产品链接`。  
随后由 OpenClaw 通过定时任务每天执行一次竞品信息抓取，并将结果写回客户飞书多维表。

这一需求下，系统需要满足：

1. 在每天固定时间自动触发数据更新。
2. 先对飞书表中的 `产品链接` 做格式化处理。
3. 对重复的 `产品链接` 做去重。
4. 找出待更新的数据行。
5. 对每一条待更新数据，根据信息采集逻辑抓取竞品信息并回写。
6. 如果商品页明确表现为已下架、商品不存在或区域不可售，则在飞书表中把 `商品状态` 标记为 `已下架/区域不可售`，并停止该行后续抓取。

### 4.2 需求二：OpenClaw 关键词搜索竞品并写入飞书

客户会在 OpenClaw 中输入类似下面的需求：

`帮我查询/收集关键字为“east egg”的 7 日内销量大于 200 的 TK 商品数据`

这一需求下，系统需要满足：

1. 能识别用户输入中的 `关键词` 和 `7 日销量阈值`。
2. 数据源固定为 FastMoss。
3. 搜索结果固定按 `近7天销量` 排序。
4. 需要翻页穷举搜索结果。
5. 只处理 `近7天销量` 大于用户输入阈值的商品。
6. 能得到 `0` 条或 `N` 条候选竞品数据。
7. 对每一条候选商品，先判断该商品是否已存在于飞书表中。
8. 如果商品已存在，则直接跳过，不更新、不重写该条数据。
9. 如果商品不存在，则先把 `SKU-ID` 写入飞书表。
10. 再复用统一的信息采集逻辑，把其他自动维护字段补全并写回飞书。
11. 对于通过关键词搜索新增写入的商品，需要在 `备注` 中写明来源，格式为：`通过搜索关键字：{关键词}`。
12. 如果新商品后续详情抓取时发现商品页明确不可访问，则保留已写入的 `SKU-ID`、`产品链接`、`备注`，并额外把 `商品状态` 标记为 `已下架/区域不可售`。

## 5. 统一信息采集逻辑

### 5.1 待更新判断

如果飞书表中的某一行数据，在自动维护字段中存在 `1` 个或 `N` 个字段为空，则该行属于待更新数据。

自动维护字段仅限以下 14 个：

- `产品链接`
- `SKU-ID`
- `图片`
- `标题`
- `节日`
- `卖家`
- `前台截图`
- `价格`
- `fastmoss价格`
- `Fastmoss截图`
- `昨日销量`
- `近7天销量`
- `近90天销量`
- `记录日期`

除上述字段外，其他字段为空，不影响待更新判断。

补充规则：

- 如果某一行 `商品状态` 已经是 `已下架/区域不可售`，则该行默认不再进入后续抓取。
- `商品状态` 不属于自动维护字段，因此它本身为空或非空都不影响 14 个自动维护字段的完整性判断。

### 5.2 产品链接格式化与去重

`产品链接` 的处理规则沿用之前版本的 cleanup 逻辑：

1. 先检查该行是否属于空行；如果是空行，则直接删除整行。
2. 将链接标准化为标准 TikTok PDP URL。
3. 将标准化结果写回 `产品链接`。
4. 按标准化后的 `产品链接` 去重。
5. 遇到重复时保留首条，删除后续重复行。

这里的“空行”定义为：

- 以 cleanup 阶段读取到的该行原始字段集为准；
- 如果该行所有字段都没有有效值，则该行视为空行；
- 只有 `产品链接` 为空但其他字段仍有值的行，不属于空行，不应在 cleanup 阶段直接删除。

### 5.3 商品定位逻辑

每条数据的商品定位规则如下：

1. 如果该行存在 `产品链接`，则优先使用 `产品链接` 访问商品页面。
2. 如果该行没有 `产品链接`，但存在 `SKU-ID`，则通过 `SKU-ID` 拼接商品链接后访问商品页面。
3. 如果人工修改了 `产品链接` 或 `SKU-ID`，需要同时人工清空 `商品状态`，让该行重新进入抓取流程。

### 5.4 信息采集与写回逻辑

统一的信息采集逻辑如下：

1. 定位商品页并抓取 TikTok 商品信息。
2. 如果商品页明确表现为商品已下架、商品不存在或区域不可售，则不再进入 FastMoss，并把 `商品状态` 写为 `已下架/区域不可售`。
3. 只有在商品页可正常访问，且当前仍缺 `fastmoss价格 / Fastmoss截图 / 昨日销量 / 近7天销量 / 近90天销量` 其中至少一个字段时，才进入对应的 FastMoss 商品详情页。
4. 打开 FastMoss 商品详情页后先截图。
5. 将截图写入 `Fastmoss截图`。
6. 再抓取 `fastmoss价格`、`昨日销量`、`近7天销量`、`近90天销量`。
7. 仅对当前为空的自动维护字段执行写回，不覆盖已有内容。
8. 只有当本次确实有字段被写入时，才同步刷新 `记录日期`。

“商品页明确不可访问”的判定范围包括但不限于：

- 页面明确提示商品不可访问
- 页面明确提示商品不存在或已下架
- 页面明确提示 `Product not available in this country or region` 一类区域不可售信息

`昨日销量` 的当前采集策略：

- 通过 FastMoss 日期控件选择日期，不通过手动输入日期文本。
- 优先尝试选择昨天日期。
- 如果昨天日期不可选，则尝试选择前天日期。
- 如果前天日期也不可选，则将 `昨日销量` 写为 `-1`。

FastMoss 详情页截图时机示例：

- 商品 `1732042172195308576`
- 详情页地址：`https://www.fastmoss.com/zh/e-commerce/detail/1732042172195308576`
- 打开该页面后先截图，再获取对应时间段的销量信息

写回内容至少包括：

- `产品链接`
- `SKU-ID`
- `图片`
- `标题`
- `节日`
- `卖家`
- `前台截图`
- `价格`
- `fastmoss价格`
- `Fastmoss截图`
- `昨日销量`
- `近7天销量`
- `近90天销量`
- `记录日期`

其中：

- `产品链接` 的标准化回写属于前置 cleanup 阶段，不属于信息采集阶段的覆盖范围。
- `记录日期` 只在本次确实有字段被写入时同步刷新。
- 如果某一行在 cleanup 完成后 14 个自动维护字段都已完整，则不再进入信息采集阶段。
- 如果商品被判定为不可访问，则本次只写 `商品状态=已下架/区域不可售`，不再抓取 FastMoss，也不刷新 `记录日期`。
- `商品状态` 用于后续跳过不可访问商品，不属于自动维护字段。
- 如果当前缺失字段只落在 TikTok 侧字段，而 FastMoss 相关字段已完整，则本次不再重复进入 FastMoss。
- `备注` 不参与待更新判断。
- `备注` 只在关键词搜索新增商品时写入 `通过搜索关键字：{关键词}`。

## 6. 最终交付形式

最终交付形式是 OpenClaw skills。

skills 需要能够覆盖以下两类场景：

### 6.1 定时任务场景

skills 需要支持在 OpenClaw 中配置定时任务，并在定时任务触发后完成以下动作：

1. 读取客户飞书多维表中的数据。
2. 在链接预处理阶段删除空行。
3. 对 `产品链接` 做格式化处理。
4. 对重复 `产品链接` 做去重。
5. 获取待更新的数据；自动维护字段已完整的行直接跳过。
6. 按统一信息采集逻辑逐条抓取并更新飞书表。
7. 已被标记为 `已下架/区域不可售` 的商品默认直接跳过，除非人工先清空 `商品状态`。

### 6.2 对话触发场景

skills 需要支持用户在 OpenClaw 对话中输入类似下面的指令：

`帮我查询/收集关键字为“east egg”的 7 日内销量大于 200 的 TK 商品数据`

在该场景下，skills 需要完成以下动作：

1. 从用户输入中识别 `关键词` 和 `7 日销量阈值`。
2. 通过 FastMoss 搜索竞品商品。
3. 固定按 `近7天销量` 排序并翻页穷举结果。
4. 只保留 `近7天销量` 大于阈值的商品。
5. 判断候选商品是否已存在。
6. 已存在商品直接跳过，不更新该条数据。
7. 不存在商品先写入 `SKU-ID` 和按 `SKU-ID` 拼接并标准化后的 `产品链接`。
8. 再复用统一信息采集逻辑补全其他自动维护字段。
9. 对新写入商品在 `备注` 中标记 `通过搜索关键字：{关键词}`。
10. 如果新商品详情页明确不可访问，则把 `商品状态` 标记为 `已下架/区域不可售`，后续默认跳过。

## 8. 验收口径

当前版本至少需要满足以下结果：

1. 客户可以继续使用当前飞书多维表；当前表结构中包含 `fastmoss价格` 和 `商品状态` 字段。
2. 文档中的自动维护字段固定为 `产品链接 / SKU-ID / 图片 / 标题 / 节日 / 卖家 / 前台截图 / 价格 / fastmoss价格 / Fastmoss截图 / 昨日销量 / 近7天销量 / 近90天销量 / 记录日期`，`商品状态` 不属于这 14 个自动维护字段。
3. 待更新判断只基于自动维护字段，`商品状态`、`备注` 和其他客户自行维护字段不参与待更新判断。
4. 客户在飞书表中新增 `产品链接` 后，OpenClaw 能按天定时更新竞品信息。
5. 定时更新前，skills 能在链接预处理阶段识别空行并删除整行。
6. 这里的空行定义为 cleanup 读取到的原始字段中所有字段都没有有效值；如果只有 `产品链接` 为空但其他字段仍有值，则不属于空行。
7. 定时更新前，skills 能对 `产品链接` 做格式化并对重复链接去重。
8. 重复 `产品链接` 去重时保留首条，删除后续重复行。
9. 当有 `产品链接` 时，系统能基于 `产品链接` 抓取数据。
10. 当没有 `产品链接` 但有 `SKU-ID` 时，系统能基于 `SKU-ID` 拼接商品链接并抓取数据。
11. 如果商品页明确显示商品不可访问、已下架、不存在或区域不可售，系统会把 `商品状态` 标记为 `已下架/区域不可售`，并停止该行后续抓取。
12. 被标记为 `已下架/区域不可售` 的商品，后续默认直接跳过；如果人工改了 `SKU-ID` 或 `产品链接`，需要同时人工清空 `商品状态` 才能重新抓取。
13. 只有在商品页可正常访问且仍缺 FastMoss 相关字段时，系统才会进入 FastMoss 商品详情页，并先截图、再抓取 `fastmoss价格` 与销量信息、再写入 `Fastmoss截图`。
14. 如果当前仅缺 TikTok 侧字段而 FastMoss 相关字段已完整，系统不会重复进入 FastMoss。
15. 采集完成后，系统能把对应自动维护字段写回飞书多维表，并更新 `记录日期`。
16. 用户可以在 OpenClaw 中通过自然语言输入关键词检索需求。
17. 关键词搜索固定使用 FastMoss 数据源，固定按 `近7天销量` 排序，并翻页穷举。
18. 系统能根据关键词与 `7 日销量阈值` 获取 `0` 条或 `N` 条候选竞品商品。
19. 关键词搜索命中已存在商品时，会直接跳过，不更新、不重写该条数据。
20. 关键词搜索命中的新商品会先写入 `SKU-ID`，再补全其他自动维护字段。
21. 关键词搜索新增写入的商品会在 `备注` 中标记 `通过搜索关键字：{关键词}`。
22. 如果关键词搜索新增商品在详情抓取阶段被判定为不可访问，系统仍会保留该商品已有的 `SKU-ID`、`产品链接`、`备注`，并写入 `商品状态=已下架/区域不可售`。
