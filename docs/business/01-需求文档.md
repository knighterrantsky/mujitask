# 需求文档

更新时间：`2026-04-03`

## 1. 文档目的

本文件只记录业务需求本身，包括目标、边界、字段范围、阶段划分、需求变更和版本管理。  
本文件不承载任何部署参数、访问令牌、表格地址、浏览器配置或 OpenClaw 运行配置。

## 2. 当前业务目标

当前项目的业务目标是：

1. 以飞书多维表格作为主入口，读取现有 TikTok 竞品记录。
2. 基于每行 `产品链接` 抓取 TikTok 商品页阶段一字段。
3. 把抓取结果直接回写到当前飞书记录。
4. 提供一个辅助入口，对 `产品链接` 做规范化和去重。

## 3. 当前正式能力

### 3.1 主能力

- `tiktok_feishu_batch_sync`
- 从飞书表读取记录，判断阶段一字段是否缺失，逐条抓取、上传附件并回写当前行。

### 3.2 辅助能力

- `tiktok_product_link_cleanup`
- 读取飞书表中的 `产品链接`，统一格式化为标准 TikTok PDP URL，并删除重复整行。

### 3.3 调试能力

- `tiktok_product_to_feishu`
- 输入单个 TikTok URL，构建飞书字段预览。
- 仅用于调试，不作为正式客户入口。

## 4. 当前硬约束

- 不新增字段。
- 不删除字段。
- 不修改字段定义。
- 允许对 `产品链接` 做规范化后回写。
- 允许基于规范化后的 `产品链接` 判重并删除重复整行。
- 阶段一正式写回只能发生在现有飞书字段内。
- 所有图片类字段必须上传图片附件本身，不能只写链接。
- `tiktok_feishu_batch_sync` 必须是飞书表驱动入口，不是 `product_urls[]` 批量插入接口。

## 5. 当前字段范围

### 5.1 输入字段

- `产品链接`
- `关键词`
- `备注`

### 5.2 阶段一目标字段

- `SKU-ID`
- `图片`
- `标题`
- `节日`
- `卖家`
- `前台截图`
- `价格`
- `记录日期`

### 5.3 阶段二预留字段

- `Fastmoss截图`
- `昨日销量`
- `近7天销量`
- `近90天销量`
- `开售时间`
- `第一波高峰期`
- `第二波高峰期`
- `价格趋势`

说明：

- 阶段二当前只保留需求定义，不纳入本轮正式交付。

## 6. 当前正式流程

### 6.1 辅助预处理流程

`tiktok_product_link_cleanup` 业务要求：

1. 读取飞书记录。
2. 提取 `产品链接` 中的 TikTok `product_id`。
3. 规范化为标准 TikTok PDP URL。
4. 回写原字段 `产品链接`。
5. 基于规范 URL 去重。
6. 保留首条，删除后续重复整行。

### 6.2 主流程

`tiktok_feishu_batch_sync` 业务要求：

1. 读取当前飞书表 / 视图中的记录。
2. 只处理 `产品链接` 非空且已规范化的记录。
3. 如果同一规范 URL 还有重复记录，则跳过并提示先 cleanup。
4. 如果阶段一目标字段全部有值，则跳过。
5. 如果阶段一目标字段存在空缺，则执行“抓取 -> 上传附件 -> 写回当前行”。
6. 只补当前缺失字段，但只要发生写回就刷新 `记录日期`。

## 7. 当前交付范围

### 7.1 已纳入正式交付

- 飞书表驱动的 TikTok 阶段一抓取。
- `产品链接` 规范化与去重。
- 浏览器打开 TikTok 商品页。
- 抓取阶段一字段。
- 上传 `图片` 和 `前台截图` 到飞书附件。
- 单条成功后立即写回飞书。
- 失败记录和失败原因汇总。

### 7.2 当前不纳入正式交付

- FastMoss 阶段二自动化。
- 新字段设计。
- 表结构变更。
- 宿主机部署细节。
- OpenClaw 或本地 skill 配置细节。

## 8. 验收口径

业务层当前版本至少需要满足：

- 能读取飞书表真实记录。
- 能把 `产品链接` 格式化并回写到原字段。
- 能按格式化后的 URL 去重并删除重复行。
- 能正确判断记录是否需要阶段一抓取。
- 能通过浏览器抓取阶段一字段。
- 能把 `图片` 和 `前台截图` 写成飞书附件。
- 能在单条成功后立即更新当前飞书行。
- 能在失败时返回失败记录和失败原因。
