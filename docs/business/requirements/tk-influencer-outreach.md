# 达人建联表需求

更新时间：`2026-05-28`

- 入口任务：`tiktok_influencer_outreach_sync`
- 触发方式：定时任务或手动触发
- 来源表：达人建联表
- 数据来源：FastMoss 商品详情页「商品关联视频」接口、FastMoss 视频详情 overview 接口
- 接口事实来源：[../../reference/fastmoss-known-interfaces.md](../../reference/fastmoss-known-interfaces.md)

## 1. 流程定位

达人建联表用于跟踪表内达人是否已经为指定商品发布带货视频，并持续刷新该达人对该 SKU 的视频数量、聚合播放量和代表视频链接。

系统按飞书表中的 `SKUID` 和 `达人ID` 进行商品视频分页、视频索引沉淀和播放量刷新。商品视频分页窗口以飞书远程表字段为事实来源，不新增本地游标表；人工修改飞书 `检查时间` 或其他流程写入该字段导致的窗口异常属于业务接受风险。

## 2. 当前已澄清需求

1. 表中 `SKUID` 按当前业务理解等同于 FastMoss 商品 `product_id`。
2. 表中 `达人ID` 当前按 FastMoss/TikTok `unique_id` 使用，不按 FastMoss 数值型 `uid` 使用。
3. 匹配 FastMoss 商品关联视频记录时，使用 `row.unique_id` 或 `row.author.unique_id` 与飞书 `达人ID` 比对。
4. 系统读取字段：`SKUID`、`达人ID`、`视频链接`、`视频发布时间`、`检查时间`、`播放量`、`视频数量`、`更新时间`。
5. `SKUID` 为空或 `达人ID` 为空的行直接跳过。
6. 已有 `视频链接` 的达人行不参与商品视频分页窗口计算，但仍参与播放量和视频数量刷新。
7. `检查时间` 只服务于“无视频链接达人是否已检查”，不表示播放量刷新时间。
8. `更新时间` 表示飞书行数据本次实际发生写入变化。
9. `/api/goods/v3/video` 只用于发现 SKU 关联视频，并沉淀 `product + creator + video` 关系。
10. `/api/video/overview` 是每条视频播放量的最终统计来源；不使用商品视频列表接口的 `play_count` 作为最终播放量口径。
11. 同一 SKU + 达人可能存在多条视频，`播放量` 写入全部已知视频 overview 播放量总和。
12. `视频数量` 写入该 SKU + 达人全部已知视频数量；数量变化时更新。
13. `视频链接` 写入该 SKU + 达人播放量最高的视频 URL；最高播放视频变化时可以覆盖。
14. `视频发布时间` 写入该 SKU + 达人最早发布视频的发布时间；字段已有值时后续不更新。

## 3. 采集与匹配逻辑

建议流程：

1. 读取达人建联表字段：`SKUID`、`达人ID`、`视频链接`、`视频发布时间`、`检查时间`、`播放量`、`视频数量`、`更新时间`。
2. 跳过 `SKUID` 为空或 `达人ID` 为空的行。
3. 按 `SKUID` 分组，避免按飞书行逐条请求 FastMoss 商品视频列表。
4. 每个 SKU 下，只使用 `视频链接` 为空的达人行计算商品视频分页窗口。
5. 如果该 SKU 下参与窗口计算的无视频链接达人行都没有 `检查时间`，本次对该 SKU 走全量商品视频分页，FastMoss 参数使用 `d_type=0`。
6. 如果该 SKU 下参与窗口计算的无视频链接达人行存在 `检查时间`，取其中最早的 `检查时间 - 1 天` 作为 `start_date`，任务当天作为 `end_date`，执行增量商品视频分页。
7. `force_full=true` 或显式指定 `start_date/end_date` 时，任务参数优先于飞书字段。
8. 商品视频分页使用 `pagesize=5`。
9. 将 FastMoss 商品视频列表返回记录归一化为：
   - `product_id`
   - `creator_unique_id`
   - `video_id`
   - `create_date`
   - `video_url`
10. 商品视频分页完成后，按本轮飞书表中的 `SKUID + 达人ID`，从数据库检索该 SKU 下该达人的全部已知视频。
11. 对检索到的每条 video 调用 `/api/video/overview` 获取播放量，并持久化一条视频播放量快照。
12. 按 SKU + 达人聚合视频数量、总播放量、最高播放视频和最早发布时间。
13. 商品视频列表抓取失败时，该商品对应行不写任何检查结果，等待后续重试。

## 4. 视频链接生成结论

FastMoss 商品关联视频接口本身未直接返回完整 TikTok 视频 URL，但已返回生成官方 TikTok 视频链接所需的稳定字段：

- `video_id`
- `row.unique_id` 或 `row.author.unique_id`

因此当前可以基于已有接口字段生成目标视频链接：

```text
https://www.tiktok.com/@{unique_id}/video/{video_id}
```

示例：

```text
https://www.tiktok.com/@heidiann__/video/7641370220849859854
```

当前判断：在不要求从 FastMoss 返回体直接读取 URL 的前提下，已有商品视频列表字段足够生成飞书 `视频链接` 字段。

`视频链接` 的写回口径为该 SKU + 达人播放量最高的视频 URL。最高播放视频由 `/api/video/overview` 播放量决定，播放量相同时按视频发布时间和 `video_id` 稳定选择。

## 5. 聚合字段写回口径

1. `播放量`：该 SKU + 达人全部已知视频的 overview 播放量总和。
2. `视频数量`：该 SKU + 达人全部已知视频数量；如果昨天为 11 条，今天新发现 1 条，则更新为 12。
3. `视频链接`：该 SKU + 达人播放量最高的视频 URL；最高播放视频变化时可以覆盖原值。
4. `视频发布时间`：该 SKU + 达人最早发布视频的发布时间；只在字段为空时写入，后续不更新。
5. `检查时间`：只在无 `视频链接` 的达人行完成本次检查时写回；已有 `视频链接` 的达人行不更新该字段。
6. `更新时间`：只在本次实际写入任一字段时更新。
7. 若聚合结果与飞书现有值一致，且无需写入 `检查时间`，则不写该行。

## 6. 分行写回规则

1. 无 `视频链接` 的达人行：
   - 完成本次检查后写回 `检查时间`。
   - 找到视频时写回 `视频链接`、`视频发布时间`、`播放量`、`视频数量`。
2. 已有 `视频链接` 的达人行：
   - 不更新 `检查时间`。
   - 刷新 `播放量`、`视频数量`。
   - 最高播放视频变化时更新 `视频链接`。
   - 不更新已有 `视频发布时间`。
3. 商品视频列表或视频 overview 抓取失败时，不写入不完整聚合结果，等待后续重试。

## 7. 可行性验证结论

当前需求已完成关键可行性验证：

1. FastMoss 商品关联视频接口可按商品查询视频列表。
2. 接口支持全量分页参数 `d_type=0`。
3. 接口支持自定义日期区间。
4. 接口支持按发布时间倒序排序。
5. 接口支持分页，并返回 `data.total`。
6. 接口返回 `unique_id`，可用于匹配飞书 `达人ID`。
7. 接口返回 `video_id` 和 `create_date`，可用于生成视频链接和写入视频发布时间。
8. `/api/video/overview` 返回单条视频维度播放量，可用于持久化视频播放量快照并进行达人聚合。

## 8. 仍需实现阶段确认的点

1. 飞书 `SKUID` 是否在所有实际行中都稳定等同于 FastMoss `product_id`。
2. 飞书 `达人ID` 是否在所有实际行中都按 TikTok/FastMoss `unique_id` 填写。
3. `create_date` 只有日期粒度；如果后续需要精确到时分秒，需要另找视频详情来源。
4. 多取 1 天后可能重复命中历史视频，依赖数据库视频关系去重和播放量快照追加避免重复统计。
5. 人工修改飞书 `检查时间` 可能造成分页窗口过大或过小，该风险按当前需求接受。

## 9. 验收口径

1. 只有 `视频链接` 为空的达人行参与商品视频分页窗口计算。
2. 已有 `视频链接` 的达人行仍参与播放量和视频数量刷新。
3. 当参与窗口计算的无视频链接达人行都没有 `检查时间` 时，SKU 使用 `d_type=0` 全量分页。
4. 当参与窗口计算的无视频链接达人行存在 `检查时间` 时，SKU 使用最早 `检查时间 - 1 天` 到任务当天的日期区间分页。
5. `播放量` 最终来源是 `/api/video/overview`，不是 `/api/goods/v3/video` 的 `play_count`。
6. `播放量` 写入 SKU + 达人维度全部已知视频播放量总和。
7. `视频数量` 写入 SKU + 达人维度全部已知视频数量，并在数量变化时写回。
8. `视频链接` 写入 SKU + 达人维度播放量最高的视频 URL，最高播放视频变化时可以覆盖旧值。
9. `视频发布时间` 取 SKU + 达人维度最早发布视频时间；字段已有值时后续不更新。
10. `检查时间` 只在无视频链接达人行完成检查时更新；已有视频链接达人行不更新该字段。
11. `更新时间` 只在本次实际写入任一字段时更新。
12. 本阶段不新增本地游标表，不修改代码、contract、schema 或测试。
