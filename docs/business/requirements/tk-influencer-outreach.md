# 达人建联表需求

更新时间：`2026-05-22`

- 入口任务：`tiktok_influencer_outreach_sync`
- 触发方式：定时任务或手动触发
- 来源表：达人建联表
- 数据来源：FastMoss 商品详情页「商品关联视频」接口
- 接口事实来源：[../../reference/fastmoss-known-interfaces.md](../../reference/fastmoss-known-interfaces.md)

## 1. 流程定位

达人建联表用于跟踪表内达人是否已经为指定商品发布带货视频。

系统按表中每行的商品 ID 和达人 ID 进行检查。如果该达人已经发布对应商品的视频，则认为该行完成建联，并把对应视频链接、视频发布时间和检查时间写回表中。

## 2. 当前已澄清需求

1. 表中 `SKUID` 按当前业务理解等同于 FastMoss 商品 `product_id`。
2. 表中 `达人ID` 当前按 FastMoss/TikTok `unique_id` 使用，不按 FastMoss 数值型 `uid` 使用。
3. 匹配 FastMoss 商品关联视频记录时，使用 `row.unique_id` 或 `row.author.unique_id` 与飞书 `达人ID` 比对。
4. `视频链接` 已有内容的行直接跳过，不覆盖。
5. `SKUID` 为空或 `达人ID` 为空的行直接跳过。
6. 同一商品同一达人匹配到多条视频时，优先使用发布时间最早的一条。
7. `视频发布时间` 写入被选中视频的 `create_date`。
8. 首次检查使用近 90 天回溯。
9. 后续增量检查按 `检查时间` 往前多取 1 天到任务触发日期作为查询窗口，避免日期边界漏数。
10. 如果某个商品的视频列表抓取失败，该商品对应行不更新 `检查时间`。

## 3. 采集与匹配逻辑

建议流程：

1. 第一次读表，收集所有非空 `SKUID`，并按商品去重。
2. 第二次读表，读取行级字段：`SKUID`、`达人ID`、`视频链接`、`视频发布时间`、`检查时间`。
3. 跳过 `SKUID` 为空、`达人ID` 为空或 `视频链接` 已有内容的行。
4. 按商品聚合待检查行，避免按飞书行逐条请求 FastMoss。
5. 对首次检查商品，抓取近 90 天商品关联视频列表。
6. 对已有检查时间的商品，抓取 `检查时间日期 - 1 天` 到任务触发日期的商品关联视频列表。
7. 将 FastMoss 返回的视频记录归一化为：
   - `product_id`
   - `creator_unique_id`
   - `video_id`
   - `create_date`
8. 对每行按 `SKUID == product_id` 且 `达人ID == creator_unique_id` 匹配视频。
9. 匹配到多条时，选择 `create_date` 最早的视频。
10. 匹配成功时写入 `视频链接`、`视频发布时间` 和 `检查时间`。
11. 未匹配但该商品抓取成功时，只更新 `检查时间`。
12. 商品抓取失败时，该商品对应行不写任何检查结果，等待后续重试。

## 4. 视频链接生成结论

FastMoss 商品关联视频接口本身未直接返回完整 TikTok 视频 URL，但已返回生成官方 TikTok 视频链接所需的稳定字段：

- `video_id`
- `row.unique_id` 或 `row.author.unique_id`

因此当前可以基于已有接口字段生成目标视频链接：

```text
https://www.tiktok.com/@{unique_id}/video/{video_id}
```

示例：

```text
https://www.tiktok.com/@heidiann__/video/7641370220849859854
```

当前判断：在不要求从 FastMoss 返回体直接读取 URL 的前提下，已有接口数据足够生成飞书 `视频链接` 字段，不需要额外接口才能完成基础回写。

## 5. 可行性验证结论

当前需求已完成关键可行性验证：

1. FastMoss 商品关联视频接口可按商品查询视频列表。
2. 接口支持近 90 天回溯。
3. 接口支持自定义日期区间。
4. 接口支持按发布时间倒序排序。
5. 接口支持分页，并返回 `data.total`。
6. 接口返回 `unique_id`，可用于匹配飞书 `达人ID`。
7. 接口返回 `video_id` 和 `create_date`，可用于生成视频链接和写入视频发布时间。

## 6. 仍需实现阶段确认的点

1. 飞书 `SKUID` 是否在所有实际行中都稳定等同于 FastMoss `product_id`。
2. 飞书 `达人ID` 是否在所有实际行中都按 TikTok/FastMoss `unique_id` 填写。
3. `create_date` 只有日期粒度；如果后续需要精确到时分秒，需要另找视频详情来源。
4. 多取 1 天后可能重复命中历史视频，依赖 `视频链接` 已有则跳过的规则避免覆盖。

## 7. 验收口径

1. 当行内 `SKUID` 和 `达人ID` 均有值，且 FastMoss 商品关联视频中存在同商品同 `unique_id` 视频时，系统写入视频链接、视频发布时间和检查时间。
2. 当行内 `视频链接` 已有内容时，系统不覆盖该行。
3. 当同一达人同一商品有多条视频时，系统选择发布时间最早的一条。
4. 当商品抓取成功但未匹配到该达人视频时，系统只更新检查时间。
5. 当商品抓取失败时，系统不更新该商品对应行的检查时间。
6. 当行内 `SKUID` 或 `达人ID` 为空时，系统跳过该行。
