# 关键词搜索选品写入需求

更新时间：`2026-06-10`

- 入口任务：`search_keyword_selection_products`
- 触发方式：OpenClaw 对话输入
- 业务主表：`TK选品收集`
- 共用口径：表结构、自动维护字段、非自动维护字段以 [../business-requirements.md](../business-requirements.md) 为准。

## 1. 流程定位

客户会在 OpenClaw 中输入自然语言需求，让系统按关键词搜索 TikTok / FastMoss 商品，并把符合选品条件的新商品写入 `TK选品收集`。

该流程负责“关键词发现商品 -> 过滤候选商品 -> 新增选品种子行 -> 采集新行详情”。它只对新插入的选品行触发后续数据采集；已存在商品默认跳过，不覆盖、不刷新已有行。

## 2. 客户需求

系统需要满足：

1. 能识别用户输入中的关键词和选品筛选条件。
2. 基于 FastMoss 关键词搜索商品候选列表。
3. 在插入飞书前完成候选过滤。
4. 将符合条件且不存在的商品新增到 `TK选品收集`。
5. 新增选品行需要保留关键词来源，便于后续追溯。
6. 已存在商品直接跳过，不重复新增，也不覆盖已有字段。
7. 仅对本次新增成功的选品行触发后续数据采集。

## 3. 信息采集与回写逻辑

### 3.1 关键词搜索

1. 用户通过 OpenClaw 输入自然语言需求。
2. 系统识别 `关键词`，并识别或补齐选品筛选条件。
3. 系统基于 FastMoss 搜索商品候选列表。
4. 默认筛选条件为：
   - `近7天销量 >= 500`
   - 商品价格满足 `price range maximum >= 10.99`
5. 用户输入中显式给出销量或价格阈值时，以用户输入为准。
6. 销量筛选由参数语义驱动，不为 `近7天销量` 和 `总销量` 维护两套独立业务流程：
   - 用户未显式说明销量口径时，默认使用 `近7天销量 >= 500`。
   - 用户显式说明 `总销量`、`累计销量` 或 `total sales` 时，使用 `total_sales_threshold` / `min_sold_count` 作为总销量筛选条件。
   - 显式总销量筛选不自动叠加默认 `近7天销量 >= 500`。
7. 一次 FastMoss 关键词搜索只能选择一个销量主口径作为排序和分页提前停止依据；`近7天销量` 和 `总销量` 不能同时作为搜索排序参数提交。用户同时提出 7 日销量和总销量条件时，系统应先要求用户选择一个主搜索口径，不得静默组合两个销量排序口径。
8. 总销量筛选使用 FastMoss 搜索的总销量倒序参数口径，并按总销量阈值过滤候选商品；该排序、过滤和分页提前停止能力属于通用 FastMoss 搜索参数能力，而不是选品流程内的独立分支逻辑。
9. 总销量筛选不改变选品表的默认价格口径；用户未显式给出价格阈值时，仍默认使用 `price range maximum >= 10.99`。
10. 当 FastMoss 返回价格区间时，价格筛选使用该区间的最大值。
11. 价格筛选只用于候选商品过滤，不作为分页提前停止条件。

### 3.2 新增选品种子行

候选商品写入 `TK选品收集` 前，系统按以下规则处理：

1. 以商品身份判断候选商品是否已存在于 `TK选品收集`。
2. 已存在商品直接跳过，不覆盖已有行，不触发后续详情采集。
3. 不存在商品按 `insert_if_absent` 语义新增选品种子行。
4. 新增种子行写入以下字段：
   - `商品ID`
   - `商品链接`
   - `关键词`
   - `备注`
   - `记录日期`
5. `备注` 固定写入：`通过搜索关键字：{关键词}`。
6. `商品链接` 写入标准化 TikTok 商品链接。

### 3.3 新行详情采集

1. 只有 `seed_status=success` 的新增选品行会进入后续详情采集。
2. 已跳过的已有商品不触发采集 fan-out。
3. 新增选品行复用 `选品采集需求` 中的行级采集逻辑。
4. 后续采集补齐 `TK选品收集` 的自动维护字段。
5. 如果新商品在详情采集阶段被判定为链接不可访问、下架或区域不可售，系统保留已写入的种子字段，并按选品采集口径写入 `商品状态`。

### 3.4 FastMoss 风控处理

FastMoss 搜索安全验证属于异常风控处理，不属于正常采集固定步骤。

1. 正常 FastMoss 搜索不额外等待验证码图片或滑块元素。
2. 当 FastMoss 搜索接口返回安全验证信号时，系统先刷新登录态并重试原始搜索请求。
3. 刷新后仍触发风控时，才进入 FastMoss 浏览器风控解除流程。
4. 风控解除成功判据必须回到原始搜索接口验证，不能用打开任意商品详情页作为搜索风控解除成功依据。
5. FastMoss cookies 不得进入 summary、log 或 result。

## 4. 最终交付形式

最终交付形式是 OpenClaw skills 和 Runtime workflow。

skills 需要支持用户在 OpenClaw 对话中输入类似下面的指令：

`帮我按关键词 east egg 搜索 7 日销量大于 500 且价格大于 10.99 的 TK 商品，写入 TK选品表`

或：

`帮我按关键词 east egg 搜索总销量大于 200 且价格大于 10.99 的 TK 商品，写入 TK选品表`

在该场景下，skills 需要完成以下动作：

1. 从用户输入中识别关键词、销量口径、销量阈值和价格阈值。
2. 通过 FastMoss 搜索商品候选列表。
3. 在插入前按参数化筛选条件过滤候选商品。
4. 判断候选商品是否已存在于 `TK选品收集`。
5. 已存在商品直接跳过。
6. 新商品写入 `商品ID`、`商品链接`、`关键词`、`备注` 和 `记录日期`。
7. 对新写入商品触发选品采集。

## 5. 验收口径

1. 用户可以在 OpenClaw 中通过自然语言输入关键词选品需求。
2. 未显式给出筛选条件时，默认按 `近7天销量 >= 500` 和 `price range maximum >= 10.99` 过滤候选商品。
3. 用户显式给出 7 日销量、总销量或价格阈值时，以用户输入为准。
4. 用户显式给出总销量阈值时，系统使用总销量参数口径筛选候选商品，并且不自动叠加默认 7 日销量阈值。
5. 用户同时显式给出总销量阈值和 7 日销量阈值时，系统不得同时提交两个销量排序口径；应先要求用户选择一个主搜索口径。
6. 总销量筛选不取消默认价格筛选；用户未显式给出价格阈值时，仍默认按 `price range maximum >= 10.99` 过滤候选商品。
7. 系统先过滤 FastMoss 搜索列表，再写入 `TK选品收集`。
8. 已存在商品直接跳过，不覆盖、不刷新、不触发详情采集。
9. 新增选品行写入 `商品ID`、`商品链接`、`关键词`、`备注` 和 `记录日期`。
10. 关键词来源备注格式为 `通过搜索关键字：{关键词}`。
11. 只有新增成功的选品行触发后续数据采集。
12. 后续详情采集复用 `选品采集需求` 的字段、状态和写回口径。

## 6. 变更影响边界

本流程文档可独立变更以下内容：

1. 自然语言输入中关键词和筛选条件的识别口径。
2. FastMoss 搜索、排序、分页和候选过滤规则。
3. 新增商品的判重规则和种子行写入规则。
4. 搜索来源备注格式。
5. 本流程的交付动作和验收口径。

以下变更需要同步更新主需求文档：

1. `TK选品收集` 表结构变化。
2. 自动维护字段或非自动维护字段的共用口径变化。
3. 新增或废弃正式业务流程。
4. 本流程变更影响选品采集流程。
