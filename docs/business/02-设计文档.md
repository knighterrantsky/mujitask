# 设计文档

更新时间：`2026-04-03`

## 1. 设计目标

当前设计目标是把业务收敛成两个正式入口：

1. `tiktok_product_link_cleanup`
2. `tiktok_feishu_batch_sync`

其中：

- cleanup 是辅助预处理入口
- batch sync 是正式主入口

## 2. 对外接口总览

| 接口名 | 类型 | 角色 | 功能 |
| --- | --- | --- | --- |
| `tiktok_product_link_cleanup` | CLI task | 正式接口 | 清洗 `产品链接`、规范化 URL、删除重复整行 |
| `tiktok_feishu_batch_sync` | CLI task | 正式接口 | 从飞书表读取待处理记录，抓取 TikTok 阶段一字段并回写 |
| `tiktok_product_to_feishu` | CLI task | 调试接口 | 输入单个 TikTok URL，构建飞书字段预览 |

## 3. 顶层返回结构

所有 CLI task 顶层固定返回：

```json
{
  "status": "success",
  "run_id": "run_xxx",
  "result": {},
  "error": null,
  "run_file": "runtime/cli_runs/xxx.json",
  "steps_file": "runtime/cli_runs/steps/xxx.json",
  "signals_file": "runtime/cli_runs/signals/xxx.json",
  "artifacts_dir": "runtime/artifacts/run_xxx"
}
```

读取顺序：

1. 先看 `status`
2. 失败时读 `error`
3. 成功时读 `result.data`

## 4. 接口一：`tiktok_product_link_cleanup`

### 4.1 输入

```json
{
  "table_url": "https://my.feishu.cn/base/appXXX?table=tblXXX&view=vewXXX",
  "access_token_env": "FEISHU_ACCESS_TOKEN",
  "url_field_name": "产品链接",
  "run_mode": "canary",
  "trace_id": "optional"
}
```

### 4.2 输出

`result.data` 重点字段：

- `summary`
- `items`
- `settings`

### 4.3 功能

- 读取飞书表中的 `产品链接`
- 提取 `product_id`
- 规范化为标准 TikTok PDP URL
- 回写原字段 `产品链接`
- 基于规范 URL 删除重复整行

### 4.4 典型状态

- `preview`
- `updated`
- `delete_preview`
- `deleted`
- `invalid_url`
- `skipped_empty`

## 5. 接口二：`tiktok_feishu_batch_sync`

### 5.1 输入

```json
{
  "table_url": "https://my.feishu.cn/base/appXXX?table=tblXXX&view=vewXXX",
  "access_token_env": "FEISHU_ACCESS_TOKEN",
  "url_field_name": "产品链接",
  "profile_ref": "local-chrome",
  "run_mode": "canary",
  "max_records": 0,
  "retry_attempts": 3,
  "retry_delay_sec": 3.0,
  "record_delay_sec": 2.0,
  "record_delay_jitter_sec": 2.0,
  "pause_every": 5,
  "pause_sec": 8.0
}
```

### 5.2 输出

`result.data` 重点字段：

- `summary`
- `items`
- `failed_items`
- `settings`

### 5.3 功能

- 从飞书表读取现有记录
- 判断阶段一字段是否存在空缺
- 对需要处理的记录逐条打开 TikTok 商品页
- 抓取商品字段、生成本地图片、上传附件、更新当前飞书行

### 5.4 正式写入字段

- `SKU-ID`
- `图片`
- `标题`
- `节日`
- `卖家`
- `前台截图`
- `价格`
- `记录日期`

### 5.5 典型状态

- `preview`
- `updated`
- `failed`
- `skipped_not_cleaned`
- `skipped_duplicate_needs_cleanup`
- `skipped_completed`
- `skipped_empty`
- `invalid_url`

## 6. 流程设计

### 6.1 cleanup workflow

当前 workflow：`tiktok_product_link_cleanup_v1`

步骤：

1. `load_records`
2. `normalize_urls`
3. `delete_duplicate_rows`
4. `write_back_normalized_urls`
5. `emit_summary`

### 6.2 batch sync workflow

当前 workflow：`tiktok_feishu_batch_sync_v1`

步骤：

1. `load_records`
2. `filter_target_rows`
3. `process_target_rows`
4. `emit_summary`

### 6.3 单条处理顺序

对于每条需要抓取的记录，固定顺序为：

1. 抓取
2. 上传附件
3. 写回当前飞书行

## 7. 模块设计

### 7.1 Task 层

职责：

- 暴露 task 名称与描述
- 根据 `run_mode` 选择 workflow
- 按 step id 调用对应 flow 方法

当前 task：

- `TikTokProductLinkCleanupTask`
- `TikTokFeishuBatchSyncTask`
- `TikTokProductToFeishuTask`

### 7.2 Workflow 层

职责：

- 定义执行步骤
- 定义 preconditions / postconditions
- 定义 step outputs 与 artifacts

当前 workflow builder：

- `build_tiktok_product_link_cleanup_workflow`
- `build_tiktok_feishu_batch_sync_workflow`
- `build_tiktok_product_to_feishu_workflow`

### 7.3 Feishu API 层

主要对象和方法：

- `FeishuAPIError`
- `parse_table_url(table_url)`
- `FeishuBitableClient.list_records`
- `FeishuBitableClient.list_all_records`
- `FeishuBitableClient.update_record`
- `FeishuBitableClient.create_record`
- `FeishuBitableClient.delete_record`
- `FeishuBitableClient.upload_media`

### 7.4 TikTok 商品抓取层

主要方法：

- `extract_tiktok_product_id`
- `normalize_tiktok_product_url`
- `fetch_tiktok_product_record`
- `fetch_tiktok_product_record_via_browser`
- `extract_tiktok_product_from_html`
- `download_tiktok_product_main_image`
- `build_feishu_bitable_fields`
- `build_feishu_bitable_record`
- `infer_tiktok_product_holiday`

### 7.5 TikTok -> 飞书同步层

主要方法：

- `run_tiktok_product_link_cleanup`
- `run_tiktok_feishu_batch_sync`
- `load_cleanup_records`
- `normalize_cleanup_records`
- `delete_cleanup_duplicates`
- `write_back_cleanup_records`
- `build_cleanup_summary`
- `load_batch_sync_records`
- `filter_batch_sync_rows`
- `process_batch_sync_rows`
- `build_batch_sync_summary`
- `_prepare_writable_fields`
- `_build_stage1_success_fields`
- `_process_batch_sync_row`
- `_is_stage1_completed`
- `_missing_stage1_field_names`
- `_select_stage1_write_fields`

### 7.6 数据模型

- `TikTokProductRecord`
  - 表示 TikTok 商品抓取结果及本地附件路径
  - 提供 `to_dict()` / `from_dict()`

### 7.7 OpenClaw 运行时包装层

OpenClaw 运行层不直接暴露全部业务参数，而是通过 skill 包装脚本固定以下值：

- `url_field_name = 产品链接`
- `profile_ref = local-chrome`

Skill 本地持久化配置只保留：

- `INSTALL_DIR`
- `TABLE_URL`
- `FEISHU_ACCESS_TOKEN`
