## 6. 当前阶段需求重写
### 6.1 总体流程
1. 人工在飞书表维护 `产品链接` 等输入信息。
2. 程序先对 `产品链接` 做格式化，并将格式化结果回写到原字段。
3. 程序基于格式化后的 `产品链接` 判重，保留每组中的首条记录，删除重复行。
4. 阶段一只根据当前阶段目标字段是否为空决定是否采集。
5. 如果某条记录的阶段一目标字段全部已有值，则直接跳过，不再重复抓取。
6. 如果某条记录的阶段一目标字段存在一个或多个空缺，则抓取该条记录，并在单条成功后立即上传附件、立即写回飞书，不做整批缓存。
7. 阶段二通过浏览器打开 FastMoss，补充销量和趋势信息。
6. 当前阶段冻结的是表结构，不是字段内容。

### 6.2 阶段一：TikTok 商品页采集
- 数据来源为 TikTok 前台商品页。
- 技术路线为浏览器驱动，不再把 `requests + HTML` 作为正式方案。
- 阶段一只关注当前阶段目标字段：
  - `SKU-ID`
  - `图片`
  - `标题`
  - `节日`
  - `卖家`
  - `前台截图`
  - `价格`
  - `记录日期`
- 阶段一的触发条件如下：
  - 如果上述字段全部已有值，则跳过该记录
  - 如果上述字段中有一个或多个为空，则抓取该记录
- 阶段一只要对某条记录执行写回，就必须同时更新 `记录日期`，即使该字段原本已有值
- 阶段一执行方式改为“读一条、抓一条、写一条”：
  - 不再先把整批采集结果缓存到内存后统一写回
  - 单条记录采集成功后，立即上传 `图片`、`前台截图`
  - 单条记录上传成功后，立即写回当前记录
- 阶段一写回时，仍然只允许写入当前阶段目标字段，不写任何状态字段或中间字段。
  - 空缺字段需要补写
  - `记录日期` 需要每次写回都刷新
- 进入商品页后需要：
  - 等待页面进入稳定态
  - 提取 `SKU-ID`
  - 提取 `标题`
  - 提取 `卖家`
  - 提取 `价格`
  - 识别并保存商品主图到 `图片`
  - 生成整页截图并保存到 `前台截图`
  - 基于标题关键词识别 `节日`
  - 写入 `记录日期`
- 在进入采集前，允许先把 `产品链接` 格式化后回写到原字段。
- `图片` 和 `前台截图` 必须上传图片文件本身到飞书附件字段，不能只写图片地址。
- 如果单条记录采集失败，需要在运行结果中报告失败记录和失败原因。

### 6.3 阶段二：FastMoss 采集
- 数据来源为 FastMoss 页面。
- 目标是为同一条记录补充：
  - `Fastmoss截图`
  - `昨日销量`
  - `近7天销量`
  - `近90天销量`
  - `开售时间`
  - `第一波高峰期`
  - `第二波高峰期`
  - `价格趋势`
- 当前表中没有 `近28天销量` 字段，因此本阶段不记录 28 天销量。
- 当前阶段只保留这部分需求定义，不要求本轮开发完成。
- `Fastmoss截图` 和 `价格趋势` 必须上传图片文件本身到飞书附件字段，不能只写图片地址。

### 6.4 链接整理需求
- 链接整理是当前阶段正式需求的一部分。
- 处理规则如下：
  - 从 `产品链接` 中提取 TikTok `product_id`
  - 统一格式化为 `https://www.tiktok.com/shop/pdp/{product_id}`
  - 将格式化结果回写到现有 `产品链接` 字段
  - 以格式化后的 URL 为主键判重
  - 每组重复记录保留首条，删除后续重复整行
- 当前阶段不新增 `标准化链接` 之类的新字段，直接在现有字段内完成格式化和去重。
